### 配置文件

#### index.php

```php
<?php
    phpinfo();
?>
```

#### nginx.conf

```nginx
#允许进程数量，建议设置为cpu核心数或者auto自动检测，注意Windows服务器上虽然可以启动多个processes，但是实际只会用其中一个
worker_processes  auto;

events {
    #单个进程最大连接数（最大连接数=连接数*进程数）
    #根据硬件调整，和前面工作进程配合起来用，尽量大，但是别把cpu跑到100%就行。
    worker_connections  1024;
}

http {
    #文件扩展名与文件类型映射表(是conf目录下的一个文件)
    include       mime.types;
    #默认文件类型，如果mime.types预先定义的类型没匹配上，默认使用二进制流的方式传输
    default_type  application/octet-stream;

	#sendfile指令指定nginx是否调用sendfile 函数（zero copy 方式）来输出文件，对于普通应用，必须设为on。
    # 如果用来进行下载等应用磁盘IO重负载应用，可设置为off，以平衡磁盘与网络IO处理速度。
    sendfile        on;

     #长连接超时时间，单位是秒
    keepalive_timeout  65;

    # =============== 日志配置 ===============
    # 定义日志格式（可选，使用 main 格式）
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    # 设置一个 default server 返回 444（拒绝）或 404，用于捕获非法请求
    server {
        listen 80 default_server;
        server_name _;
        return 444;  # 关闭连接
    }
    
    server {
        listen 80;
        server_name www.gsdn-php8.com;
        root E:/work/code/php/gsdn-php8;
        index index.php index.html;

        location / {
            try_files $uri $uri/ /index.php?$query_string;
        }

        # 处理 .php 文件
        location ~ \.php$ {
            fastcgi_pass 127.0.0.1:9000;
            fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
            include fastcgi_params;
            fastcgi_hide_header X-Powered-By;
        }
    }

}
```

#### php.ini

将 `php.ini-development` 复制为 `php.ini` 即可

#### hosts

```
127.0.0.1 www.gsdn-php8.com
```

### 启动

**启动 php**

在 `php` 安装目录执行如下命令

```
php-cgi.exe -b 127.0.0.1:9000 -c php.ini
```

**启动 nginx**

在 `nginx` 安装目录直接启动 `nginx` 即可, 关闭 `cmd` 窗口即可关闭 `nginx` 

```
nginx.exe
```

### 配置

> 注意: xdebug 需要绝对路径

```
zend_extension=D:/program/php/php8/php-8.3.29-nts-Win32-vs16-x64/ext/php_xdebug.dll
```

### 启用配置

如下几个配置需要开启

![](/images/2026-07-01-php-开发环境搭建/009.png)

添加如下配置即可

```
; 1. 加载扩展 (Windows 下通常是 xdebug.dll)
; 请确保路径正确，例如：E:/env/php/ext/php_xdebug.dll 或只是 xdebug
zend_extension=D:/program/php/php8/php-8.3.29-nts-Win32-vs16-x64/ext/php_xdebug.dll
; 2. 运行模式 (核心)
; 推荐组合：debug (断点调试) + develop (友好的错误页面)
; 如果需要性能分析，可额外添加 profile，但平时建议关闭以提升速度
xdebug.mode = debug,develop

; 3. 连接设置
; 指向你的 IDE (VS Code / PhpStorm) 所在的地址
xdebug.client_host = 127.0.0.1
; Xdebug 3 默认端口是 9003 (旧版是 9000)，请确保 IDE 监听此端口
xdebug.client_port = 9003

; 4. 触发机制 (关键性能优化)
; yes: 每次请求都尝试连接 (方便但稍慢)
; trigger: 仅在带有特定 Cookie/GET 参数时连接 (推荐生产或高性能需求)
; 对于本地开发，推荐设为 'yes' 以获得无缝体验，或者 'trigger' 配合浏览器插件
xdebug.start_with_request = yes

; 5. 日志与排错 (如果连接失败，开启此项查看原因)
; 建议先注释掉，调试通后再关闭，避免磁盘写满
; xdebug.log = "E:/work/code/php/gsdn-php8/storage/logs/xdebug.log"
; xdebug.log_level = 7

; 6. 其他优化
; 防止远程服务器误连，只允许本地发起
xdebug.discover_client_host = 0
; IDE 密钥 (可选，如果你的 IDE 配置了特定的 IDE Key，如 PHPSTORM 或 VSCODE)
xdebug.idekey = "PHPSTORM"
```
